## Box CLI v3.0.0 released

  Today, we released `v3.0.0` of the [Box CLI][2022-02-01_17].  This release has several enhancements, most notably - addition of OAuth 2.0 support.

{/* more */}

In addition to CLI enhancements and documentation changes, we also released a brand new [YouTube video][2022-02-01_18] that shows using our Box CLI Quick Start to get started with the CLI in less than
five minutes!

## Developer Documentation Changes

* Updated the [Box CLI Quick Start Guide][2022-02-01_19] to use the OAuth 2.0 Authentication method
* Moved the [JWT setup steps][2022-02-01_20] to an appendix

## New Features and Enhancements

* Add support for login with OAuth ([#240][2022-02-01_3])
* feat: support as-user flag for bulk files and when token is present ([#270][2022-02-01_4])
* Add support for `copyInstanceOnItemCopy` field for metadata templates  ([#239][2022-02-01_5])
* Add support note to the `mdfilter` equality check in search ([#253][2022-02-01_6])
* Add support for Box Sign API ([#258][2022-02-01_7])
* Add support Metadata Query API ([#259][2022-02-01_8])
* fix: `folder:collaborations:add` make `role` a required flag (`SDK-1070`) ([#261][2022-02-01_9])
* Minor dependencies upgrade
* Add new API for files and file versions under retention ([#250][2022-02-01_10])
* Adding support for sign request ([#258][2022-02-01_11])
* Support Metadata Query API ([#259][2022-02-01_12])
* chore: enforce conventional commits ([#268][2022-02-01_13])
* feat: support as-user flag for bulk files and when token is present ([#270][2022-02-01_14])

## Breaking Changes

* Drop support for Node 10
* Insensitive language changes ([#247][2022-02-01_1], [#252][2022-02-01_2])

## Bug Fixes

* fix: folder:collaborations:add make role a required flag ([#261][2022-02-01_15])
* fix: Fixed shared-links delete example. ([#262][2022-02-01_16])

## Where to get support

Should you have any issues or need further guidance, please post a request to
our [developer forum][2022-02-01_0] for any help needed. You can also connect with the SDK
team on the respective GitHub repository.
