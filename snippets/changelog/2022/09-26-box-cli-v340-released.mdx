## Box CLI v3.4.0 released

  ### New Features and Enhancements

* Add `DryRun` mode in `User Deprovision` example script ([#392][2022-09-26_1]) ([`584a30e`][2022-09-26_2])
* Add `redirect_url` and `declined_redirect_url` to Sign Request ([#395][2022-09-26_3]) ([`261b7d2`][2022-09-26_4])
* Add progress bar for file download and bulk command ([#376][2022-09-26_5]) ([`68359c7`][2022-09-26_6])
* create destination path during download ([#393][2022-09-26_7]) ([`40881dd`][2022-09-26_8])
* native credentials storage for Windows with a new library ([#385][2022-09-26_9]) ([`a6918aa`][2022-09-26_10])

### Bug Fixes

* correctly resolve home `dir` in folders download ([#398][2022-09-26_11]) ([`86d3230`][2022-09-26_12])
* Fix logs in `Users Deprovisioning` script ([#381][2022-09-26_13]) ([`c85f77b`][2022-09-26_14])
* Fix problem with `keytar` library ([#394][2022-09-26_15]) ([`1979f01`][2022-09-26_16])

### Note

* To migrate storing Box CLI environments to the system credential storage, execute the following command in the terminal ([#295](https://github.com/box/boxcli/issues/295))

    `box configure:environments:update`
